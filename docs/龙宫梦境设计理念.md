---

## 一、设计理念

### 1.1 核心思想

> **九儿在对话中不用分心写日记，梦境在后台自动完成记忆整理。**

将记忆整理工作从对话中剥离，交给定时任务自动完成。对话中九儿专注服务用户，只在**重大项目进展**时手动记录重要内容（双保险原则）。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **自动化优先** | session 对话自动拉取 → daily → MEMORY.md，无需手动 |
| **双保险** | 重大项目进展时手动写入 daily，防止自动遗漏 |
| **安全操作** | 原始文件不删除，仅归档重叠部分到回收站 |
| **简要汇报** | 执行后汇报不超过 15 行，不冗长 |
| **模块化** | 两个核心功能独立，可单独调用 |

## 二、核心工作流

```
原始 session 对话（自动拉取）
         ↓
九儿提炼 → 写入 daily memory（YYYY-MM-DD.md）
         ↓
优化 MEMORY.md（查重/合并/补充）
         ↓
更新 USER.md（偏好 + 羁绊提炼）
```

### 2.1 每日任务（02:00）

```
sessions_list → sessions_history → 提炼 → memory/YYYY-MM-DD.md
```

- 拉取活跃 session 的对话历史
- 提取 7 类信息：事件、决策、技术、教训、待办、羁绊、用户偏好
- 同一天多条 session 合并为一个 daily 文件
- 输出提炼报告 + 偏好 JSON

### 2.2 每周任务（周日 02:15）

```
读取 daily 文件 → 查重去重 → 重建 MEMORY.md → 更新 USER.md → 通知天哥
```

- 查重：时间线同日去重、教训关键词去重、待办前缀去重
- 补充：从 daily 提取新待办和新教训
- 重建：按 12 章节结构生成新 MEMORY.md
- 更新 USER.md：偏好表格 + 羁绊章节

## 八、注意事项

1. **操作前必先读取所有文件**，不跳过
2. **原始 daily 文件不删除**
3. **MEMORY.md 时间线最多 20 条**
4. **教训去重基于前 30 字符**
5. **每次优化后更新「最后更新」时间**
6. **重大项目进展时手动写入 daily（双保险）**
7. **汇报不超过 15 行**
