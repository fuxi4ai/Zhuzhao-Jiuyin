## 一、背景与目标

体系数据已齐（recap.db 31 表 + market_data.db 行情/成交额/美股锚 + 龙鱼标的库），但结论散在 N 个表和文档里，盘前/盘后要人肉拼。本日报把五层信息（市场→周期情绪→信号→机会→风险）做成**每日一张 HTML**，30 秒读完核心，3 分钟钻完细节。

**用户**：Doctor（盘前 5 分钟 / 盘后复盘）。
**成功标准**：打开后不需要再查任何表就能回答——今天市场什么水位？哪条线在兑现？有没有新机会/新风险？

## 二、内容需求 × 数据源映射（五区块）

| 涨/跌家数 | `stock_daily` 按日聚合 pct_chg 正负 | ✅（⚠️ 仅 20260603 后全市场口径，GOTCHA 红线） |
| 总成交额 | `market_amount_daily.total_trillion`（正源） | ✅ |
| 涨停 / 跌停数 | `daily_market.limit_up / limit_down` | ✅（同口径红线） |
| 美元:人民币汇率 | 🆕 `market_data.db.fx_daily`（tushare fx_daily，USDCNH 离岸为主，Doctor 终端跑 `fetch_fx.py`，已批） | ✅ 待首次回填 |

呈现：顶部一行五卡片 + 涨跌家数红绿分布条；成交额卡片同时显示**对应 K_cap 容量档**（联动区块 3）。

| 字段 | 数据源 | 状态 |

### 区块 3 · 信号与容量（体系核心）

| 字段 | 数据源 | 状态 |
|---|---|---|
| 产业逻辑信号（当日新增/活跃，gap_level≥3 高亮） | `industry_signals`（keyword/etf_anchor/info_gap_level/confidence） | ✅ |
| 市场兑现信号 | `gap_status / gap_desc / date_realized / excess_peak`（closure_engine） | ✅ |
| 主线热度清单 | `gap_position_overlay` 的 gap_hot_themes（主线:活跃数:最高gap） | ⏸️ v0.2 待 --apply |
| **CAP 容量仪表** | K_day（当日涨幅>+1% 主线数，theme_etf_daily）vs K_cap（经验表插值：1万亿→1-2 / 2→3-4 / 3→5-6） | ✅ 口径已定 |
| 美股锚隔日映射 | `us_anchor_daily`，完整 L1 逻辑：对齐口径（A股 D 日用最近一个收盘<D 09:30 的美股交易日，显示 us_date + stale_days，周一 stale=3 降权重）；echo 类锚显示隔夜超额 + 三态判定——**先行**（锚超额转正而 A股线未启动=跨市场 gap 加分）/ **共振**（锚与 A股线同向=兑现可信度↑）/ **背离**（A股 closing 但锚 20 日超额转负=警示，进区块 5）；±2% 异动标记（ALM 放宽 ±4%）；thermo 类锚（TSM/AAPL/LLY/LMT/GS）只展示温度不参与判定 | ✅ 数据已回填，口径=隔日映射设计 §二§三 |

呈现：
- **容量仪表盘**：K_day / K_cap 指针 + 三态标签（有空位 / 满载 / 虹吸）——这是全页的"天气预报"；
- 主线兑现状态矩阵：18 主线 × {gap_desc 定性, 20 日超额迷你走势 sparkline, 美股锚隔夜箭头}，点击主线展开该线信号明细表（可按 gap_level / 状态筛选）。

### 区块 4 · 机会提示 + 标的 + 仓位建议

| 字段 | 数据源 | 合成逻辑 |
|---|---|---|
| 机会提示 | 兑现初期/潜伏期信号 × 容量有空位/虹吸态 × 美股锚同向 | 三条件交集自动生成，按 gap_level 排序 |
| 对应标的 | `industry_signals.target` + 龙鱼-标的分析库 `records/*.json`（分数/评级/thesis） | 标的卡片显示龙鱼分+评级+一句话 thesis；库内无评分标的标"未评分" |
| 仓位建议 | 小鲍 `dim4_trade_plan.position_band/risk_pref`（基准档）+ 容量状态修正 + 该线兑现阶段 | **开放问题 #2**：三源合成权重 Doctor 拍板；v1 先并列展示三个口径不合成 |

呈现：机会卡片流（线 → 信号摘要 → 标的 chips → 三口径仓位并列）。
**固定脚注：体系内信号聚合，非投资建议。**

### 区块 5 · 风险提示

自动聚合五类，按严重度排序：
1. **美股锚背离**（A股 closing 但锚 20 日超额转负，如 LITE −20.7% vs 光模块 +43%）
2. **逾期未启动**（gap_desc ⚠️ 类，逻辑存疑清单）
3. **容量满载且单族虹吸**（新线挤不进 / 轮动风险）
4. **估值极限标的**（龙鱼估值维 ≤3/15 且处主线内，如 PB>95% 分位）
5. **regime 红线**（持仓/关注标的净利下滑触零容忍）

呈现：红黄两级横幅，每条可点开看依据数据。

## 三、格式与美学要求

- **单文件 HTML**（数据构建时嵌入为内联 JSON，离线可开，零后端）；CDN 仅允许 ECharts（图表）。
- **深色星河系**视觉，与 `龙鱼标的库.html`（AI4ME/龙鱼五力-outputs）同语言：深底、高对比数据色（涨红跌绿按 A 股惯例）、克制的发光强调；中文为主，等宽数字字体。
- **数据诚实铁律（Doctor 2026-06-10 裁定）**：缺数 = 诚实标注"该源停更于 X 日 / 待回填"，**禁止占位符、禁止编数据、禁止用旧值冒充当日值**；每个区块自带 vintage 角标。
- **信息层级**：首屏 = 五卡片 + 容量仪表 + 周期情绪（30 秒层）；下钻 = 矩阵/明细表（3 分钟层）。不许首屏出现表格。
- **交互**（纯前端）：主线矩阵点击展开明细；信号表按 gap_level/状态/主线筛选排序；sparkline 悬停显示数值；机会/风险卡片折叠展开；顶部日期显示数据 vintage（各源各自的最新日期，**不假装同一天**）。
- 移动端可读（Doctor 手机看盘前），单列降级布局。

## 四、技术方案

```
tools/gen_daily_report.py
  读 recap.db + market_data.db + 龙鱼records + overlay 输出
  → 校验各源 vintage、缺数标"待更新"
  → 注入模板 templates/daily_report.html（Jinja 或字符串替换）
  → 输出 AI4ME/烛照九阴-outputs/
       烛照九阴日报_YYYYMMDD.html      ← 根目录只留最新一份
       archived/烛照九阴日报_*.html    ← 生成新报时把旧报移入，永不删除（删除权归 Doctor 手动）
```

- 生成全程沙箱可跑（只读两库）；行情/汇率等拉数仍走既有 Doctor 终端管线。
- 更新机制：v1 手动跑；v2 挂 `recap-kejian-daily-ingest` 调度尾部（数据全更后自动出报）。
- 留档规则（Doctor 裁定）：根目录=最新，往期自动移 `archived/`，CC 永不删任何一期。

## 五、非目标（v1 明确不做）

- 不做实时盘中刷新（日频）；不做后端/数据库直连（嵌入式数据）；不做个股全列表（只展示信号相关标的）；不做仓位三口径自动合成（开放问题 #2 拍板前并列展示）；不内嵌美股锚 L2 校准结果（校准完成后 v2 进）。
