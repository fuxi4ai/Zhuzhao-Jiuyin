## 1. gangtise (钢体) — 消息面渠道

| 项目 | 内容 |
|------|------|
| **定位** | 金融"消息面"渠道 |
| **状态** | ⏳ 约 1 周后上线正式版 |
| **提供数据** | 财经新闻、政策公告、行业催化、突发事件 |
| **对应需求** | Dim3 消息催化 (news_catalysts) |
| **对应表** | `dim3_sentiment_tech.news_catalysts` |
| **对接优先级** | P1（消息面直接影响情绪判断） |

### 对接方案

```
gangtise API
    ↓
消息提取 → 关键词 / 行业标签 / 影响方向 (利好/利空)
    ↓
匹配日期 → 关联到对应复盘记录
    ↓
更新 dim3_sentiment_tech.news_catalysts
```

### 应用场景

| 场景 | 说明 |
|------|------|
| 催化事件关联 | 某天的涨停潮，是因为 gangtise 上的哪条消息？ |
| 消息→情绪验证 | 利好消息当天，情绪是否确实复苏？ |
| 消息→板块映射 | gangtise 上的"固态电池突破" → 关联到"新能源电池"板块 |

## 2. tushare pro — 行情数据库

| 项目 | 内容 |
|------|------|
| **定位** | 中国 A 股专业行情数据 API |
| **状态** | ✅ 已接入 (Token 验证通过) |
| **提供数据** | 日线行情、涨跌停统计、成交量、板块指数、个股行情 |
| **对应需求** | 量化情绪周期 7 指标 + 回测验证 |
| **对应表** | `cycle_quant`, `signal_backtest`, `tushare_stats` |
| **对接优先级** | P0（量化系统必需） |

### 已接入数据

| tushare 接口 | 复盘数据库字段 | 状态 |
|-------------|---------------|------|
| `limit_list_d` | limit_up/limit_down/consecutive_limit | ✅ 200/350 条 (57%) |
| `moneyflow_hsgt` | north_flow_billion | ✅ 300 条 |
| `index_daily` | 大盘指数/涨跌幅 | ✅ 535 条 |

### 数据成果

- **双轨一致率**: 34% → 85% (加入 tushare 数据后)
- 有效样本: 34 条 (有≥3 指标 + 小鲍标注)

### 数据映射

| tushare 接口 | 复盘数据库字段 | 说明 |
|-------------|---------------|------|
| `daily` | 次日收益率 | 回测验证 |
| `limit_list` | limit_up/limit_down | 量化指标 2、3 |
| `daily_basic` | volume_trillion | 量化指标 5 |
| `moneyflow` | north_flow_billion | 量化指标 6 (北向资金) |
| `limit_list_d` | consecutive_limit | 量化指标 1 (连板高度) |
| `index_daily` | 大盘指数 | 大盘收益率验证 |
| `ths_index` | 板块指数 | 板块回测 |

### 补全量化 7 指标

| # | 指标 | 当前状态 | tushare 来源 |
|---|------|---------|-------------|
| 1 | 连板高度 | ❌ 缺失 | `limit_list_d` |
| 2 | 涨停家数 | ✅ 52 条 | `limit_list` |
| 3 | 跌停家数 | ✅ 45 条 | `limit_list` |
| 4 | 涨跌比 | ❌ 缺失 | `daily` (计算涨跌家数) |
| 5 | 成交量 | ✅ 162 条 | `daily_basic` |
| 6 | 北向资金 | ❌ 缺失 | `moneyflow` |
| 7 | 主线持续性 | ❌ 缺失 | 需板块指数计算 |

### 对接方案

```
tushare pro API
    ↓
每日拉取 → 涨跌停/成交量/北向/连板/大盘涨跌
    ↓
更新 cycle_quant 表 (补全 7 指标)
    ↓
更新 signal_backtest 表 (回测验证)
```

| 项目 | 内容 |
|------|------|
| **定位** | 行业知识图谱 + Wiki 卡片 |
| **状态** | ✅ v2 已就绪 (932 节点/1155 关系/170 Wiki 卡片) |
| **提供数据** | 行业分类、产业链关系、板块 Wiki、上下游关联 |

| **对应表** | `dim2_sector_themes`, `sector_alias`, `yuantu_ref` |
| **对接优先级** | P1（板块标准化增强） |

### 已有工具

| 工具 | 路径 | 说明 |
|------|------|------|
| `kg-query` | dragon-spirit/ | 渊图知识图谱查询工具 |
| 渊图 v2 | 龙宫/ | 932 节点/1155 关系/170 Wiki 卡片 |

### 对接方案

```
渊图知识图谱 (932 节点)
    ↓
行业分类 → 标准化板块名称 (替代 manual 维护的 sector_alias)
    ↓
```

| 板块轮动 | 基于渊图的上下游关系预测轮动方向 |
| Dim2 增强 | 复盘中的"主线"关联到渊图行业卡片 |

## 4. 句芒行情库 — 回测验证引擎

| 项目 | 内容 |
|------|------|
| **定位** | 行情数据库 + 回测验证 |
| **状态** | 🔵 同平台对接（句芒也是火山引擎 jvsclaw） |
| **优势** | 数据库可按需开发，跨 session 通信方便 |
| **提供数据** | 个股/板块/指数的日级行情数据 |
| **对应需求** | 预案验证、回测统计、预测引擎基础 |
| **对应表** | `signal_backtest`, `prediction_log` |
| **对接优先级** | P0（回测和预测必需） |

### 对接方案

```
复盘信号 (62 条方向性)
    ↓
句芒行情库
    ↓
获取次日/3日/5日收益率
    ↓
计算胜率、平均收益、最大回撤
    ↓
signal_backtest 表填充
```

### 回测输出

```
方向性信号回测:
├── 总体胜率: ?% (需要句芒数据)
├── 按情绪阶段分组胜率
├── 按板块分组胜率
└── 信号质量排名

情绪周期验证:
├── 冰点期看多 → 次日胜率 ?%
├── 退潮期看多 → 次日胜率 ?%
└── 小鲍标注 vs 量化标注 → 谁更准？
```

## 数据库表扩展建议

```sql
-- 消息面表 (gangtise)
CREATE TABLE IF NOT EXISTS news_catalyst (
    date TEXT,
    source TEXT DEFAULT 'gangtise',
    title TEXT,
    content TEXT,
    impact TEXT,          -- 利好/利空/中性
    sector TEXT,          -- 关联板块
    created_at DATETIME
);

-- 渊图关联表
CREATE TABLE IF NOT EXISTS yuantu_link (
    sector TEXT,
    yuantu_id TEXT,       -- 渊图节点 ID
    wiki_card TEXT,       -- Wiki 卡片链接
    upstream TEXT,        -- 上游板块
    downstream TEXT,      -- 下游板块
    PRIMARY KEY (sector)
);

-- 回测结果表 (句芒)
CREATE TABLE IF NOT EXISTS signal_backtest (
    signal_date TEXT,
    target TEXT,
    next_day_return REAL,
    next_3d_return REAL,
    next_5d_return REAL,
    max_return REAL,
    max_drawdown REAL,
    outcome TEXT,
    verified_at DATETIME
);
```
