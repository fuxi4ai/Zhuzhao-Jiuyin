# 🔬 烛照九阴 · 复盘框架方法论评估

> 日期：2026-06-06 · 评估人：CC
> 范围：复盘框架在 **① A股经验提取** 与 **② 预测** 两方向的**合理性（设计）** 与 **可靠性（实测）**
> 依据：方法论文档 + 真库 `Database/烛照九阴/recap.db` 实测
> 说明：本文评估的是**方法论可靠性**，非投资建议。

---

## 〇、一句话结论

**这套框架的真实价值在"结构化复盘记忆 + 假设生成"，而不在"预测"。** 作为经验提取的脚手架，设计合理、有亮点、可靠性中等；作为预测引擎，骨架思路对、但落地全面失败——量化轨退化成常数、主观轨低于基准率、前瞻验证表为空。**把它当线索库用是对的，当预测器用是错配。** 两个方向缺的是同一件事：**闭环的样本外验证**。

| 方向 | 合理性（设计） | 可靠性（实测） |
|------|:---:|:---:|
| ① 经验提取 | 🟢 较高 | 🟡 中（单源/未标准化/未验证）|
| ② 预测 | 🟡 部分成立 | 🔴 低（当前不成立）|

---

## 一、方向①：A股经验提取

### 合理性 🟢 — 框架成立，有真亮点
- **四维度分解**（外围定价 / 行业主线 / 情绪技术 / 交易策略）是对 A股日度复盘的连贯、近乎 MECE 的拆解，结构合理。
- **info_gap_level（信息差→超额收益窗口）** 是真正有洞见的因果框架：用"明确度 / 直接性 / 传播广度"三维度量信息不对称，对应潜伏窗口长短。这是框架里最有 alpha 潜质的概念。
- **logic_type 驱动分类**（供给冲击 / 需求爆发 / 技术革新 / 涨价 / 事件 / 情绪周期）——把零散信号按底层驱动归类，是可复用的模式库思路。
- **方法论成熟度**：predictor_accuracy 的"每条预测标明谁说的"、错题本文化、渊图主从倒置（产业逻辑以独立专家纪要为准、小鲍降第二印证）——都体现了对单源偏差的警觉，方向正确。

### 可靠性 🟡 — 四个实测短板
1. **库里的"经验"目前=单一老师的复盘文本**。`industry_signals` 1231 条 **viewpoint_owner 全是小鲍、confidence 全是 P2**。渊图主从虽是对的方向，但库里尚未体现多源——现在提取的不是"A股市场经验"，是"一个复盘老师怎么讲盘"。
2. **分类法未标准化**。`logic_type` 有 27 个取值、中英混杂，光"涨价"系就有至少 5 个重复类（price_driven 270 / 涨价逻辑 30 / 涨价驱动 10 / 涨价 7 / price_increase 16）。任何按 logic_type 的聚合或学习都会失真。
3. **info_gap_level 主观且未校准未验证**。三维权重（直接性 60%）是拍定的；1231 条里 67.8% 全压在 level 2，区分度低；而且"信息差大→超额收益高"这个**核心假设从未被回测**（1231 条中 1199 条「待验证」，仅 7 条「已验证」）。
4. **关键字段填充率低**。据 `复盘数据要素提炼.md`：涨停家数、仓位建议、核心标的等覆盖率多在 20-40%。颗粒度受限，G007（NLP 提取噪音）仍未闭环。

**小结**：作为**灵感/线索库**可用且有亮点；作为"A股经验的可信结论库"还不可靠——欠的是多源化、分类标准化、和对 info_gap 假设的回测。

## 二、方向②：预测

### 合理性 🟡 — 骨架对，但有一处逻辑硬伤
- **双轨制**（小鲍主观 vs 量化客观）思路正确：用量化锚定主观判断，是好设计。
- **预测者准确率体系**：为每个源建可信度档案、和次日方向比对，方向对。
- ✅ **峰值判退潮 = 有意的逆向先验，非 bug**（2026-06-06 天哥订正）：阈值把最高分（涨停潮/天量/北向强）判为「退潮」，是"物极必反、最火热即退潮之始（反者道之动）"的逆向择时设计，符合情绪顶的经典逆向逻辑。CC 先前误判为"语义倒置"，撤回。
- ⚠️ **但缺趋势项——单调 level→stage 无法表达非单调的周期**：情绪周期在"水平"上是圈状（冰点低→复苏升→火热峰→退潮落→回低），**复苏与退潮同水平、反方向**，只测瞬时分数的单阈值函数分不出二者。"反者道之动"说的是拐点（导数），要落实需 `阶段 = f(水平, 趋势)`：峰值+仍升→主升/亢奋；峰值+滚头→退潮；中位+升→复苏；中位+落→退潮中段。否则"峰值=退潮"会在每个高广度日打顶，不分真滚头还是动量延续——而二者**随 horizon 不同**（次日动量常赢，1-2 周反转更可能），是**可回测命题**，非先验真理。
- 💡 连带推论：若量化轨有意在顶喊退潮、小鲍喊主升，则**极值处的"分歧"才是拐点信号**——此时拿"一致率"当质量 KPI 是搞反了，该统计"分歧后谁对"。

### 可靠性 🔴 — 五项实测硬伤（真库数据实锤）
1. **量化轨完全退化**。103 条 `cycle_quant` **全部 = 冰点**，total_score 最高仅 **25**（火热>21、退潮>30 在真实数据上根本达不到）。量化情绪周期**零区分度，等于一个常数**。
   → 双轨"一致率"因此无意义：实测 37.6% 的"一致"全是双方都说冰点的偶合，旧 README 宣称的 **93.2% 无法复现**。根因：北向/主线天数/连板大量缺失，分被钉死在低位；阈值（G010 调过仍）与实际 0-25 的分布不匹配。
2. **主观轨准确率低于基准率**。小鲍情绪阶段→次日方向 = **46.9%（38/81）**，而同期四大指数"上涨日"基准率 = **55.2%**。即**"无脑猜涨"都比这套阶段→方向映射准**。各阶段样本极小（过热 n=4=100% 纯属噪音，主升 n=3）。
3. **没有真正的前瞻验证**。`prediction_log`（追加式前瞻表）= **0 行**。所有"准确率"都是事后回填（in-sample backfill），且停在 **2026-05-12**（近一个月没更新）；`verify_daily_predictions.py` 已归档、每日自动验证没在跑。框架声称的"预测验证闭环"**实际未闭环**。
4. **信号回测形同虚设**。`industry_signals` 1231 条仅 32 条（2.6%）有验证状态，1199 待验证。旧 README"121 信号 54.5% 胜率"在现库无法复现；且 54.5% 对一个上涨基准率 55% 的市场而言**没有 alpha**。
5. **前视/标签泄漏风险**。emotion_stage 标签来自小鲍**事后**复盘，验证又用次日涨跌——标签本身可能已含后视；回填用同一段历史、无样本外检验，准确率被高估的方向。

**小结**：设计骨架（双轨 + 源可信度）合理，但落地失败——量化轨退化为常数、主观轨低于基准率、前瞻表为空、回测未做。**当前不具备可靠的预测能力，且阈值语义需纠错。**

## 三、跳出"好/坏"——这框架到底该怎么用

把它定位成**预测器**，会一直失望（情绪周期预测次日涨跌，本就是低信噪比问题）。它真正擅长的是两件事：
- **结构化记忆**：把每天的盘面、主线、催化、预案沉淀成可检索的库（recap_cli 已能查），供事后复盘、找相似历史情形（similar/rotation 命令）。
- **假设生成**：logic_type + info_gap 把"为什么涨"的逻辑显性化，产出**待验证假设**，而不是结论。

两个方向要变可靠，缺的是**同一件事：闭环的样本外验证**。

---

## 四、按性价比排序的改进建议

**预测方向（要么修好、要么别声称准确率）**
1. 🔴 **修量化轨区分度 + 加趋势项**：用公共行情库 `market_data.db` 补齐北向/连板/主线天数，把阈值重定到实际 0-25 分布；并把阶段改为 `f(水平, 趋势)`——峰值+仍升=主升/亢奋、峰值+滚头=退潮、中位+升=复苏、中位+落=退潮中段（落实"反者道之动"的拐点判定，而非单调水平映射）。先让 cycle_quant 不再是常数。
2. 🔴 **启用 prediction_log 做真正的前瞻记录**：每日入库当天就写下"明天预判"，次日用 market_data 验证，**严格样本外**；按源、按阶段统计，永远和基准率（≈55% 上涨）对比，不和 50% 对比。
3. 🟡 重启 `verify_daily_predictions` 的每日自动验证（已在 `_legacy/db/`）。

**经验提取方向**
4. 🟡 **标准化 logic_type 词表**（27→6-8 个规范类，中英合并），否则聚合无意义。
5. 🟡 **回测 info_gap 假设**：用 market_data 给 industry_signals 做事件后 N 日收益统计，验证"信息差大→超额收益高/窗口长"是否成立——成立则这是真 edge，不成立则降级。
6. 🟢 **落实渊图主从让经验库多源化**：现在全 P2 单源，把渊图 P1 专家逻辑纳入 viewpoint_owner，经验库才名副其实。
