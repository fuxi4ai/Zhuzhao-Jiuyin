## 0. 一句话结论

**现状是「半成品 + 接反了」：**

- **产业逻辑（结论层）** 是九儿**手工**从渊图 thesis/纪要 curate 的**一次性快照**——内容质量高，但不是管线、无契约、无增量、易漂移。
- **「产业逻辑：买入」信号** 则**根本没接渊图**——它是 `logic_discovery.py cmd_hot` 对**小鲍课件文本**的一句 SQL 正则（`supply_shock/price_driven + LIKE %涨价/缺口/停产/紧缺%`）。
- 而渊图**早就有**图谱级的信号机制（`tag_market_signals` + `signal_impact_chain`）和给龙鱼的版本化消费契约，**烛阴一样没用**。

主从倒置的决策（6-04）是对的，但只在产业逻辑层手工兑现了一半，**买入信号层还是纯小鲍**。

## 1. 渊图侧实际能提供什么（源能力盘点）

渊图是成熟 KG 系统，能产出的远多于烛阴现在取的：

| 层 | 产物 | 形态 / 置信度 |
|---|---|---|
| thesis | `index.json` 279 篇报告一行论点 | 帕米尔/外资纪要为主，P1 |
| 图谱 | `mapping/latest.json`（软链→canonical）1774 节点 / 2059 边 / 11 类关系 | P1=1035 / P2=168 |
| **信号** | `tag_market_signals.py`：给节点打 demand_surge / supply_shock / persistent_imbalance | rule+LLM，conf≥0.7，~106 信号节点 |
| **传导链** | `signal_impact_chain.py`：沿 causes/constrains 追 N 层 → 受益下游清单 + 跨信号汇总最脆弱节点 | 图谱活算 |
| 标的评分 | `yuantu_scoring.get_company_score(ts_code)` → 公司级评分 | 龙鱼五力在用 |
| **消费契约** | `coupling_contract.md`：latest.json 稳定入口 + healthcheck + 版本化防改名漂移 | **只对龙鱼，没给烛阴** |

> 关键：渊图的信号是**图谱级、去重、带 conf、可追传导链到下游节点**；标的级映射也有（yuantu_scoring）。这正是「产业逻辑→买入」需要的全套底座。

| 对象 | 提取方式 | 问题 |

## 3. 有效性评估（分三层）

### A. 产业逻辑（结论层）—— 内容有效，过程不可靠
- ✅ **质量高**：九儿读真专家纪要（P1），抓出标的偏差（磷化铟正宗=先导，非小鲍提的云南锗业）、国产未落地（载体铜箔）、去魅（华为韬定律）——这是「人脑 + P1 源」的真价值。
- ❌ **过程不可持续**：快照非管线；32 条 vs 渊图 279 报告 / 1774 节点的覆盖；无增量（渊图更新→结论层不会自动刷）。
- ❌ **无契约 = 裸奔**：龙鱼专门用 `coupling_contract` 根治「渊图改名→静默断」；烛阴是手工映射，渊图一改名/改结构，映射全失锚，且无人报警。
- ❌ **丢了渊图最值钱的部分**：渊图能到「公司×逻辑」级（impact_chain 追下游、yuantu_scoring 到 ts_code），烛阴只取到产业链级 32 条，把传导链和标的映射都丢了。

### B. 「买入信号」—— 接反了，是最弱环
- ❌ **没用渊图**：买入信号本应是「产业逻辑成立→受益标的」，渊图的 signal node(conf≥0.7) + impact_chain(受益方向) + company_score(标的) 正是为此而生；烛阴却用小鲍课件正则。
- ❌ **继承小鲍全部弱点**：单源 P2 / 1199 条待验证 / info_gap_level 手工未校准（见 `方法论评估_20260606.md`）。
- ❌ **关键词脆弱**：`LIKE %涨价%` 命中的是「讲了涨价」的文本，不是「涨价逻辑成立」的判断；噪音高（G007 未闭环）。
- ❌ **定义分裂**：「买入起点」在 `cmd_hot`（supply_shock+price_driven）与 `产业逻辑发现模块.md`（还有 demand_surge+tech_innovation 次强）两处口径不一，未统一。

### C. 主从倒置：意图对，落地只做了一半
决策（6-04）正确——小鲍退观点层第二印证、产业逻辑以渊图为准。但只在**结论层手工兑现**；**信号/买入层仍是纯小鲍、渊图没进来**。

## 4. 改进方向（用渊图已有能力，不造新轮子）

1. **立「渊图→烛阴」消费契约**（照搬龙鱼 `coupling_contract` 范式）：烛阴只读 `mapping/latest.json`（软链防漂移）+ 启动调 `healthcheck()`；渊图改名只重指软链，烛阴无感。**这是根治手工映射失锚的结构解。**
2. **产业逻辑改自动同步**：写 consumer 脚本读 latest.json，把 signal node + `signal_impact_chain` 输出落 `dim2p_supply_demand`，带渊图 confidence；渊图新报告入库→触发刷新（挂 daily-ingest 之后）。手工 curate 降级为**审校**，不再是录入主力。
3. **「产业逻辑：买入」重定义为渊图派生**：买入信号 = 渊图 signal node（demand_surge/supply_shock/persistent_imbalance, conf≥0.7）+ impact_chain 受益方向 +（可选）yuantu_scoring 标的评分；**小鲍课件降为第二印证**（看小鲍有没有同步喊、作情绪共振确认），不再是信号源。这才真正落实主从倒置。
4. **统一买入信号定义**：合并 `cmd_hot` 与 `产业逻辑发现模块` 两套口径，锚定在渊图信号类型上。
5. **补可验证性**：渊图信号带 report date，用 `market_data.db` 做「信号后 N 日受益方向收益」回测——P1 图谱信号比小鲍 P2 正则可回测性强得多。

```
现在：
  渊图 ──(九儿手工读 thesis/纪要,一次性)──▶ dim2p 32条产业逻辑   [无契约/无增量]

应该：

  小鲍课件 ──────────────────────────────────▶ 第二印证(情绪共振确认)
                                              ↓
                                       market_data 回测受益方向
```
