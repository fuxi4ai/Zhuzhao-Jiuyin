# 🌙 美股锚「隔日映射」纳入复盘 · 设计 v0

> 日期：2026-06-10 · CC ｜ 状态：⏸️ 设计稿，待 us_anchor_daily 回填后先校准再上岗
> 核心：美股 T 日收盘（北京时间 T+1 凌晨）先于 A股 T+1 开盘 4~5 小时——这半天的领先是结构性的、每天都有。把它从"感觉"变成复盘里可量化的输入。

---

## 一、定位：dim1 的主线级下沉

四维度框架里 `dim1_external_pricing` 已经在大盘级干这件事（纳指/恒生/汇率/油/BTC）。
隔日映射 = **把 dim1 从大盘级下沉到主线级**：纳指→AI硬件全体，下沉为 LITE→光模块、GEV→电力、MP→稀土的逐线映射。不另起炉灶，是 dim1 的自然延伸。

## 二、对齐口径（铁律）

- A股交易日 D 可用的美股锚信号 = **最近一个收盘时间 < D 09:30 的美股交易日**（通常为美东 D−1）。
- 节假日错位：取"最近可用"并记 `stale_days`（周一用上周五美股，stale=3 不加权重，只展示）。
- 信号量 = 锚对 SPY 的**隔夜超额**（绝对涨跌会被美股大盘β污染）；thermo 类锚（TSM/AAPL/LLY/LMT/GS）只展示不进信号。

## 三、三层用法（按上线顺序）

### L1 · 盘前快照（最先上，纯展示零风险）
每个 A股交易日早间产出一行/主线：`昨夜锚超额 | 连续超额日数 | 异动标记(|超额|≥2%)`。
- 落点：`recap_daily` 配套新表 `us_anchor_echo`（见 §四），每日复盘表模板加「美股锚隔夜」小节。
- 例：6-10 盘前应显示 `光模块: LITE 隔夜 −8.2%* 异动` ——1029→821 的回撤就不会等到盘后才看见。

### L2 · 传导校准（决定谁有资格进信号层）
**先回测再上岗**（info_gap_level 校准的教训）：逐主线回测
`锚隔夜超额(美东 T−1) → A股主线 ETF 当日超额(T)` 的命中率与弹性（echo_beta）。
- 预期分化：光模块/AI算力 传导强；机器人（TSLA叙事）中等；大宗（NEM/FCX/MP）走商品价不一定走A股映射；thermo 类无传导。
- 只有命中率显著 >50% 的主线，其隔夜异动才进 L3；其余永久停留在 L1 展示层。
- 产出：`docs/隔日映射校准表.md`，每季复检。

### L3 · 接入兑现引擎（最后上）
closure_engine / yuantu_buy_signals 加 `us_echo` 维度：
- **跨市场 gap**：锚已兑现（对 SPY 20日超额>阈值）而 A股主线 open → 信号加分（全球定价已确认，A股滞后=窗口）。
- **共振印证**：A股 closing 期间锚同向超额 → 兑现可信度↑。
- **背离警示**：A股 closing 但锚 20日超额转负 → 纯内资叙事标记，gap_hot_themes 降权。

## 四、落库设计

```
us_anchor_echo（recap.db，每日一主线一行）
  a_date TEXT          -- A股交易日
  theme TEXT           -- 主线（THEME_US 键）
  us_date TEXT         -- 所用美股交易日
  stale_days INTEGER   -- 数据陈旧度（正常1，周一3）
  anchor_kind TEXT     -- echo / thermo
  us_ret REAL          -- 锚隔夜涨跌%
  us_excess REAL       -- 锚对SPY隔夜超额%
  us_streak INTEGER    -- 锚连续超额日数
  alert INTEGER        -- |us_excess|≥2% 异动标记
```

- 🆕 `tools/us_echo.py`：`--snapshot`（L1 日更，挂 daily-ingest 尾部）/ `--calibrate`（L2 回测出校准表）。
- L3 改动放校准结果出来之后，不预写。

## 五、防呆清单

1. thermo 锚永不进信号层（写死在 anchor_kind 过滤）。
2. 美股财报日的锚异动是个股事件不是产业信号——L2 校准时财报周剔除，L1 展示加 `财报季` 备注（v1 先不做自动识别，人工注意）。
3. ALM 历史短（2025-07 起）、流动性薄，单日波动大，alert 阈值单独放宽到 ±4%。
4. us_anchor_daily 若某日缺数（信源断），echo 行照常生成但 `stale_days` 递增，不补造数据。
