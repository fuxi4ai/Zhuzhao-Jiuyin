# 补测 · 点亮扳机换 MACD 底部承接（Doctor 指令 2026-07-07 凌晨）

**问题**：点亮扳机由 现行 Y′=4日/回升5pp 换成 剑酒青丘"MACD 底部承接"（绿柱加深后首次收缩、仍在零轴下、价格确认），超额有几天？

**口径**：96 个去重暗态期（theme×dormant_since），每期取首个 MACD 底部承接触发，前向 1–20 日主线超额（对 510300，closure_engine 同源序列）。两变体：V1=主 ETF 收盘价上算 MACD（价格确认降级 close>前日close，theme_etf_daily 无 high 列）；V2=累计超额曲线上算 MACD。对照=现行点亮事件同窗口（n=20）。

**结果（逐日均值超额路径）**：
- V1 价格版（n=69）：第 1 天起即负（−0.53），20 天内从未翻正。3d 胜率 42%、5d 37.7%。**超额 = 0 天**。
- V2 超额曲线版（n=71）：围绕零轴漂，仅第 7–9 日短暂翻正（峰值 +0.40% @第8日），随后转负。3d 胜率 54.9%/中位 +0.13——噪声级。**可用超额 ≈ 0 天**。
- 对照现行扳机（n=20）：前 3 日微正（中位第1日 +0.15），之后一路走弱至 −2.7%（第15日）——与主报告 F2 一致。

**结论**：MACD 底部承接不能拯救点亮，反而更差——暗态期的主线本质处于相对弱势 regime，"底部形态"类扳机在这里是接刀器（V1 触发即负）。现行价格动量扳机反而是三者中唯一前 3 日为正的。**维持 F2 裁定（3 日短窗）不变，且不建议再往"更灵敏的底部扳机"方向找**；若要改善点亮质量，方向应是"更迟钝更确认"（如要求创新高确认），而非更早抄底。

**边界**：价格确认降级（无 high 列）会使 V1 触发略偏松；每期仅取首触发；单一 regime 样本。原始数据 macd_relit.json。
