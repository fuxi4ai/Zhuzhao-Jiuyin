# 产业信号「买入强度」复合分 · 口径草案 v0

- **日期**：2026-06-29
- **状态**：草案待审（Doctor 拍方向后起草；**未实现、未落库**）
- **目标**：把现在散在两轴的强度信息（确定性 / 隐蔽性）合成一个可排序的 **0–100 买入强度分** + 自动归档，解决「现无单一买入强度、confidence 和 info_gap 各排各的、信源档又是平的」的缺口。
- **Doctor 定调（2026-06-29）**：语义＝**确定性闸门 × 隐蔽加分**；产出＝**0–100 复合分 + 自动归档**。

---

## 一、设计原则

1. **确定性是主心骨，隐蔽性是加成**。先问「这条逻辑成不成立」，再问「别人有没有看见」。
2. **info_gap 倒挂用闸门压住**：`gap_rater` 的 info_gap 越高越隐蔽、但也越可能是模糊噪声（它给模糊措辞高分）。所以 **info_gap 只在确定性够高时才加分**；确定性不够，再隐蔽也不奖励（「没人看 + 没把握」＝废信号）。
3. **可排序 + 可读**：连续分供排序，自动档 + 四象限标签供一眼定性。

## 二、入料字段（全部 ex-ante，可在信号当下取到）

| 维度 | 字段 | 取值现状 |
|---|---|---|
| 确定性 | `yuantu_buy_signals.yuantu_confidence` | 0.75–0.98（聚 0.85–0.95） |
| 信源档 | `source_plevel` / `industry_signals.confidence` | P0/P1/P2；现库 industry 全 P2（暂平） |
| 机制厚度 | `signal_type` | supply_shock / demand_surge / persistent_imbalance，可叠 1–3 个 |
| 传导印证 | `beneficiary_count`、`echo_count`(小鲍回声) | 整数 |
| 隐蔽性 | `industry_signals.info_gap_level` | 1–5（现库 2 最多、5 仅 1 条） |

> 兑现侧 `excess_cum/excess_peak`、`gap_status` 是**事后**，不进本信号 ex-ante 总分（见 §五 Phase-2 先验）。

## 三、计分口径 v0

**① 归一化分量（各 0–1）**
- `conf_n = clamp((yuantu_confidence − 0.70) / 0.30, 0, 1)`  ——确定性
- `P = {P0:1.0, P1:0.8, P2:0.6}[source_plevel]`  ——信源档
- `type_n = {1类:0.6, 2类:0.8, 3类:1.0}[机制叠加数]`  ——机制厚度
- `corrob = min(beneficiary_count,5)/5 × 0.6 + min(echo_count,3)/3 × 0.4`  ——传导印证
- `gap_n = (info_gap_level − 1) / 4`  ——隐蔽性

**② 确定性基础分 Base（0–100）**
```
Base = 100 × ( 0.50·conf_n + 0.20·P + 0.15·type_n + 0.15·corrob )
```

**③ 隐蔽性闸门加成（最多 +20）**
```
gate  = clamp((Base − 45) / 20, 0, 1)      # Base 45→65 线性放行
bonus = gate × gap_n × 20
```
闸门含义：Base<45 隐蔽性完全不加分；Base≥65 全额放行；中间线性。

**④ 总分 + 归档**
```
买入强度 = clamp(Base + bonus, 0, 100)
强(≥75) / 中(55–74) / 弱(<55)
```
并叠加**确定×隐蔽四象限标签**（定性注释，不替代单分）：
金矿(确定·隐蔽) / 拥挤(确定·明牌) / 博彩(不确定·隐蔽) / 噪声(不确定·明牌)。

## 四、两个验算（证明闸门有效）

- **金矿**：conf 0.95 / P2 / 2 机制 / 受益 4 / echo 1 / gap 4
  → conf_n .833, corrob .61 → Base ≈ **75** → gate 1, gap_n .75, bonus +15 → **总分 ≈ 90 · 强 · 金矿** ✅
- **噪声**：conf 0.80 / P2 / 1 机制 / 受益 1 / echo 0 / gap 5（模糊冷门）
  → Base ≈ **39** → gate 0 → **bonus 0**（gap 5 不加分）→ **总分 ≈ 39 · 弱 · 博彩** ✅
  即：再隐蔽，确定性不够也压在弱档——正是要的效果。

## 五、诚实边界 / 待办

1. **权重是 v0 拍脑袋**，必须用历史 `gap_status∈(closing,closed)` 的 `excess_cum/peak` 做**一次回测校准**——看高分档是否真兑现更好；不通过则调权重/闸门，**回测前不定稿**。
2. **信源档现库 industry 全 P2、区分度=0**；`yuantu_buy_signals.source_plevel` 待核覆盖率。这一档当前形同常数，可先给小权重或并入待补。
3. **info_gap 是 gap_rater 启发式**（关键词），本身有噪声；闸门是缓解、非根治。
4. **Phase-2 先验**：可加「同 `signal_type`/`industry_chain` 的历史兑现率」作 ex-ante 微调（用别的同类信号的事后表现当本信号先验，不用自身未来数据，避免穿越）。
5. **落点（实现待批）**：建议新增列 `yuantu_buy_signals.buy_strength`（+ `buy_strength_tier`），由一个 `tools/buy_strength.py --apply` 计算落库（走 `config.connect_write`、/tmp 往返）；日报渊图卡排序由 `yuantu_confidence DESC` 改 `buy_strength DESC`。**这些都属实现，待本口径审定后再出逐文件清单。**

## v1（回测校准后 · 取代 v0 · 2026-06-29）

> 拿历史已兑现样本（`recap.db`：industry 940 条 / yuantu 70 条，按 `excess_cum`）回测，**颠覆了 v0 三处前提**。v0 上文保留留痕。

**回测三大发现**
1. **两表不可 join**：`yuantu.signal_node ∩ industry.keyword = 0`，etf_anchor 仅重叠 8 → 跨表复合分作废，**只能单表**。
2. **确定性轴废弃**：`corr(yuantu_confidence, excess_cum) = −0.19`（轻微反指，0.95+ 桶兑现反而最差）；`beneficiary_count = −0.26`。印证「渊图越笃定＝越明牌＝已 priced-in」。
3. **info_gap 是唯一稳杠杆**：level 1→4 兑现单调 +0.11/+0.16/+0.20/+0.26。

**辅助项验证**：`conf_weight` 恒 0.7（全 P2·零区分·弃）；`verify_status` 已/部分验证(+0.26/+0.33) > 待验证(+0.16)（小正·可加）；`logic_type` 结构型 tech_innovation/supply_shock(+0.25/+0.23) > price/event(+0.11/+0.12)（小正·轻 tilt）；`category` 半导体/光通信高、航天/光伏低 → **周期性·过拟合·弃**。

**v1 口径（仅 `industry_signals`，0–100）**
```
gap_n    = clamp((gap_raw − 1)/4, 0, 1)            # gap_raw = gap_rater 连续原始分(细于1-5整数)
verify_n = {已验证:1.0, 部分验证:0.8, 待验证/None:0}
logic_n  = {tech_innovation:1.0, supply_shock:1.0, demand_surge:0.5, 其余:0}
买入强度 = 100 × ( 0.75·gap_n + 0.15·verify_n + 0.10·logic_n )
强(≥60) / 中(40–59) / 弱(<40)                       # 阈值待二次分桶校准
```
- 主干 info_gap 75%、验证 15%、逻辑型 10%；用 gap_rater **连续 raw** 替代 1–5 整数，排序更细（需 gap_rater 落 `gap_raw` 列）。

**诚实边界（管理预期）**：info_gap 的 corr 仅 +0.11——**这是个弱 edge**，分数能把「隐蔽×结构型×已验证」排上来，但非高胜率保证；verify/logic 权重小、样本薄。**相对「直接按 info_gap_level DESC 排」，本套增量有限**，价值在：①连续 raw 排得更细；②纳入 verify/logic 小正项；③0–100 + 档的统一读数。是否值得实现，请 Doctor 权衡。

**`yuantu_buy_signals`**：本表无可用正预测字段（confidence/beneficiary 反指、echo 仅 +0.12）→ **v1 不给强度分**，维持现状（至多 echo_count 当弱 tiebreaker），待更好特征再议。

**落点（实现待批）**：仅 `industry_signals` 加 `buy_strength`(+`buy_strength_tier`)；`gap_rater` 落 `gap_raw` 连续列；阈值用历史分桶定；日报「机会/GAP 栏」排序 `info_gap_level DESC` → `buy_strength DESC`。`yuantu` 卡不动。
