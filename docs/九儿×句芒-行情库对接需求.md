## 一、对接背景

九儿的复盘数据库已完成 Phase 1-4（350 条复盘记录 + 情绪周期双轨制 + tushare 行情接入），目前缺少**回测验证能力**。需要句芒的行情数据来验证复盘中的方向性信号准确率。

### 九儿已有数据

| 资产 | 数量 | 说明 |
|------|------|------|
| 复盘主记录 | 350 条 | 2024-02-19 → 2026-05-06 |
| 方向性信号 | 62 条 | 看多/看空某板块或个股 |
| 情绪阶段标注 | 145 条 | 小鲍老师主观标注 |
| tushare 行情 | 200 条 | 涨跌停/北向/指数（部分） |

### 核心需求

**验证 62 条方向性信号的准确率**：
- "看多 AI 算力" → 次日 AI 算力板块涨了多少？
- "冰点期看多" vs "退潮期看多" → 哪个阶段胜率更高？
- 板块/个股级信号的平均收益率、最大回撤

## 四、回测接口定义

### 4.1 信号回测查询

**九儿的输入**（从复盘数据库提取）：
```json
{
  "signal_date": "2026-05-06",
  "signal_type": "BULL",
  "target": "AI算力",
  "target_type": "sector"
}
```

**需要的输出**（句芒行情数据）：
```json
{
  "signal_date": "2026-05-06",
  "target": "AI算力",
  "next_day_return": 2.3,
  "next_3d_return": 5.1,
  "next_5d_return": 3.8,
  "max_return": 7.2,
  "max_drawdown": -1.5,
  "outcome": "HIT"
}
```

### 4.2 批量回测

九儿会将 62 条信号打包成列表发送给句芒，句芒批量返回回测结果。

## 五、协作方式

### 方案 A：共享 SQLite（推荐）

```
句芒写入 → shared database/jumang/market-db/market_data.db
九儿读取 → JOIN 复盘信号 → 计算回测
```

**优势**：简单直接，两个 jvsclaw 共享 TOS 文件

### 方案 B：跨 session 通信

```
九儿 sessions_send → 句芒 session → 查询行情 → 返回结果
```

**优势**：实时查询，不需要文件共享

### 方案 C：CSV 导出

```
句芒导出 CSV → 上传到 TOS → 九儿下载导入
```

**优势**：最简方案，适合一次性导入历史数据
