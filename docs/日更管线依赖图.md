## 四 · 更新顺序（cron 实况 + 07:00 任务内部）

晨间定时链（北京时间，定义在 launchd / Skill 定时任务，见 `复盘工作流总览.md`）：

| 时刻 | 任务 | 归属 | 干什么 |
|---|---|---|---|
| 06:00 | recap-kejian-daily-ingest | 九儿 | 课件入库 → recap dims（recap_import / xiaobao_extractor）|
| 06:15 | market-data-daily-update | 句芒 | 公共行情 → market_data 句芒表 ★必须早于 07:00 |
| 06:30 | recap-kejian-review | 句芒 | 课件审核（只读核管线③）|
| 07:00 | **zhuzhao-market-fetch-daily-report** | 烛阴 | ②四表 + 5.x 派生回填 + 出报（见下）|

**07:00 任务内部顺序（2026-06-23 三线核实后定 · 已落 SKILL）**：

```
② fetch_theme_etf / market_amount / limit_list / us_anchor / fx → market_data 四表（/tmp 副本防 I/O）
   → 守门：验句芒 stock_daily 已到前一交易日（未就绪则跳过派生段、待重试，不空转）
   → recap.db 留 predaily 回滚锚 + /tmp 副本
   → emotion_engine_v2 --apply        （补 G013）
   → sync_buy_signals                  （渊图入库）
   → gap_rater                         （info_gap 分）
   → closure_engine --apply            （自动 apply + 变更 diff 落 docs/兑现变更_{日}.md + 通知，可回滚）
   → populate_signal_targets           （标的池）
   → recap.db 放回（只增不减校验 + integrity）
   → gen_daily_report                  （出暖色日报 + 同步 Artifact）
   → 落烛阴 log
```

**关键顺序铁律**：
1. 句芒公共行情（stock_daily 宽度）必须早于 07:00（① 06:15 在前）。
2. 情绪/渊图/兑现回填必须在 `gen_daily_report` 之前——否则日报读到旧状态（G013 根因）。
3. closure 自动 apply 但留 predaily 锚 + 变更 diff + 通知（gap_status 是 Doctor 拍板的确定性公式、非 CC 判断，详见 `通用教训.md` G-X23）。
