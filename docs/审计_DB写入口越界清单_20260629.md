# 审计：DB 写入口「越界挂载盘真盘」清单

- **日期**：2026-06-29
- **触发**：recap.db 反复损坏（G019）。chip 任务解封后，按治本方向 B 做的只读审计。
- **根因（探针实测）**：文件级 `cp`/`mv` 整库放回挂载盘**安全**（10/10 ok）；但**对挂载盘上的库直接 live `commit` 必报 `disk I/O error`（8/8）并甩下热 `-journal`** → 下一个进程开库时 journal 回放，回放成「孤儿索引 / 页双引用」结构损坏。
- **结构性问题**：「拒写挂载盘真盘」的护栏只在 `_ingest_九儿_2026-06-25/26.py` 里复制粘贴存在；其余写库者要么硬编码真盘无护栏（Tier-1），要么走 `config.RECAP_DB` 但无强制护栏（Tier-2）。护栏没集中，漏一个就再坏。

> 判定口径：`?mode=ro` / `immutable=1` 的连接 = 只读安全，已排除。「写」= 含 `commit / INSERT / UPDATE / DELETE / CREATE / ALTER`。

## 建议治本（待 Doctor 批方向后再出逐文件清单）

1. **集中护栏**：在 `config.py` 加 `connect_write(path)` 单一入口——路径落在挂载盘（`/sessions/` `/mnt/` 或真盘 `DATABASE_ROOT` 下）且未显式 `ZZJY_ALLOW_MOUNT_WRITE` 时，**拒绝写连接**（或自动重定向到 `/tmp` 工作副本）。所有写库者改走它，护栏只有一处、不会再漏。
2. **Tier-1 迁移**：4 个硬编码真盘脚本改为 `import config` + `connect_write`，删掉 `_docs()/expanduser` 自找真盘。
3. **Tier-2 收口**：写连接换 `connect_write`；去掉 `_ingest_25/26` 里复制粘贴的护栏，统一到中央。
4. **cp-back 卫生**（已是铁律，补强制）：放回前确认 /tmp 库无 `-wal/-journal`；放回用 `cp 新名 + mv 原子替换`；放回后强制 `integrity_check`（非 `quick_check`）+ 失败自动回滚。
