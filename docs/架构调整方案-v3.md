## 四、执行步骤

### Phase 1: 创建新目录结构
```bash
mkdir -p skills/ projects/ knowledge/ archive/{versions,docs,reports}/ temp/
```

### Phase 2: 移动热调用区（龙灵 → 根目录）
```bash
# 技能
mv dragon-spirit/skills/* skills/
# 项目
mv dragon-spirit/projects/* projects/
# 梦境系统并入项目
mv dragon-spirit/dream-system projects/dream-system/
```

### Phase 3: 移动知识库（龙宫 → knowledge/）
```bash
mv 龙宫/知识库/* knowledge/
# 渊图_v2 数据合并
mv data/渊图_v2 knowledge/raw/data/ 2>/dev/null || true
```

### Phase 4: 合并归档
```bash
mv dragon-palace/往期版本/* archive/versions/
mv dragon-palace/学习资料/* archive/docs/
mv dragon-palace/过期报告/* archive/reports/
```

### Phase 5: 合并脚本和配置
```bash
# 脚本合并
cp dragon-spirit/scripts/* scripts/
# 配置合并
cp dragon-spirit/config/* config/
# 报告归档
mv dragon-spirit/reports/* archive/reports/
```

### Phase 6: 清理旧目录
```bash
# 旧目录移入回收站（不直接删除）
bash dragon-spirit/scripts/recycle.sh dragon-spirit/
bash dragon-spirit/scripts/recycle.sh 龙宫/
bash dragon-spirit/scripts/recycle.sh data/  # 如果已空
```

### Phase 7: 修复引用路径

| 文件 | 修改内容 |
|------|---------|
| `skills/conch/SKILL.md` | 替换 `dragon-spirit/` → 新路径、`龙宫/` → `knowledge/`、`dragon-palace/往期版本/` → `archive/versions/` |
| `skills/kg-query/SKILL.md` | 替换 `dragon-spirit/skills/kg-query` → `skills/kg-query` |
| `MEMORY.md` | 更新架构路径引用 |
| `INDEX.md` | 重写为新结构 |
| `~/.openclaw/skills/conch` | **删除软链接**，改为在 openclaw.json skills.path 注册 workspace/skills/ |

### Phase 8: 技能注册（不依赖软链接）

在 `openclaw.json` 中添加 skills.path，指向 `workspace/skills/`，让 OpenClaw 直接扫描。
