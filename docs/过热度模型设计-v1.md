## 0. 定位与认识论前提（先把野心收住）

三条前提决定了整套设计的形状，不是免责声明，是规格：

1. **股市无真理**：混沌系统、复杂度过高，不存在稳定可预测的"明天涨跌"。
2. **边缘会被适应淘汰**：任何阶段性有效的策略会被市场学习、套利、失效（Adaptive Markets / 反身性）。→ **推论：不能用绝对阈值，必须用"相对市场自身近期基线"的极端度（滚动分位 / z-score）。** cycle_quant 的病根不在"用经验阈值"，在"绝对且静态"。
3. **只做能做的那件事**：不预测点位、不预测日期。**唯一目标 = 过热时点亮"退潮风险"灯**，作为减仓 / 收紧止盈 / 不追高的风险输入。这是可行的；其余不做。

> 一句话定位：**过热度是一个状态探测器，不是预测器**；输出是风险倾斜（前向回撤分布右移），不是择时。

---

## 1. 第一性原理：成交量是换手，不是买力

整套模型的地基。**成交量衡量的不是"谁在买"，而是筹码易手的规模——多空转换的剧烈程度。** 每一手成交都同时有买方与卖方，量的大小=转换强度，量的方向（涨/跌）只说明哪边更主动。

由此推出**高位天量 = 出货**，且只需一个条件：

- 天量必须有人供货；
- 久涨后的高位，**手里有这么多货可供的，只有低位吸筹的强手**；
- 所以**高位 × 天量 ⟹ 强手在批量交货，买方必然是新进弱手**（公众抬轿、强手供货）。

**关键纠偏（v1 相对早期讨论）**：不要用价格走势给多空"判输赢"。出货方**控制节奏不控制价格**——挂单慢慢卖、卖给强势，买盘越凶卖单越欢，价格照样放量大涨创新高、K 线照样漂亮。所以：

- ❌ "放量但价格推不动（滞涨/冲高回落）才算力竭" —— 会**漏掉放量大涨式见顶**（最主流形态），且等滞涨出现时派发已近尾声，灯太迟。
- ✅ **高位 × 天量本身就是全部信息**。价格在高位是伪装，不需要价格佐证。

> 对称锚点：**天量见天价 / 地量见地价**。缩量的正确归宿在**底部**（地量=卖盘枯竭），不在顶部。早期把"缩量新高=背离"挂在顶上，是把冰点的量能特征误安到了过热端。

---

## 2. 核心洞察：顶底在认识论上不对称

这是 v1 与一切"对称指标"设计的根本分歧，也是为什么顶能做、底只能描述。

| | 顶（过热/退潮端） | 底（冰点/启动端） |
|---|---|---|
| 推断结构 | **单条件事件**：高位×天量 | **多条件过程**：地量→催化→放量→测试→洗盘→突破→确认 |
| 为何 | "谁手里有货"是确定的，地心引力（卖压）站在你这边 | 地量只证明**卖方枯竭**，对买方是"证据缺失"非证据 |
| 上行/下行的本质 | **下跌只需没人买** | **上涨需要有人持续买 + 筹码集中** |
| 可做性 | **可探测** → 做（点亮风险灯） | 只配**状态描述** → 不做预测 |
| 形态 | 尖（情绪高潮，瞬时） | 圆（反复洗盘，过程） |

A 股散户主导的情绪结构正好长这样：**顶尖、底圆**。这也是"不能野心过大"的又一次自证——过热端做推断，冰点端只做观察。

---

## 3. 度量三原则

1. **一切相对滚动基线**：每个力都读成"相对近 N 日的分位 / z"，不用绝对数（对抗边缘衰减）。
2. **复杂留给特征，简单留给规则**：力的设计可以讲究，但合成用透明规则，**不上拟合模型 / ML**——非平稳小样本里过拟合是更大的敌人，且风险灯要敢照着减仓必须可解释。
3. **极端 / 背离 / 衰竭三态分开记**，不要糊成一个分。

## 4. 五力 → market_data.db 可算代理

把"资金量 / 上行动力 / 下行动力 / 价格高低点 / 多空转换"落成真能算的列，每个都问"相对滚动基线是否极端 / 背离 / 衰竭"。

| 力 | 含义 | 可算代理（数据源） | 过热端读数 |
|---|---|---|---|
| **资金量** | 换手/转换强度 | 成交额 `daily_market.volume_trillion`、换手率（`stock_daily.vol`/流通股）、北向 `north_flow` | **滚动天量分位** = 极端（注：过热=放量，非缩量）|
| **上行动力** | 多方扩散广度 | 涨停数 `daily_market.limit_up`、连板高度/晋级率/封板率（`tushare_limit`）、涨跌家数比、创新高家数（`stock_daily` 滚动 max）| 价创新高时是否**同步确认**；不确认=背离 |
| **下行动力** | 空方抬头 | 跌停数 `limit_down`、**炸板率**（`tushare_limit` 封板失败）、亏钱效应、高位股回撤 | 价still高但**在抬头**=衰竭前兆 |
| **价格高低点** | 位置极端 | 指数距 60/252 日前高位置、**新高家数−新低家数**背离 | 指数新高但新高家数收窄=顶背离 |
| **多空转换** | 转换的可见脚印 | **高位天量**（=转换极致）、主线高位股→低位补涨的高低切（`sector_daily`）| 放量是转换的脚印；缩量看不到转换 |

> 注：`tushare_limit`（recap.db，6000 行）是封/跌停板明细，连板、炸板、晋级率都从这里推。

## 5. 两条状态机

过热度不是一个静态分，是沿着**筹码结构演化**的状态推进。顶底各一条链。

### 顶链（做推断 → 点风险灯）
```
高位 × 天量
  → 【出货灯亮】筹码结构开始恶化（风险计量启动，≠ 见顶日期）
  → 天量之后，高位【量能再也放不大】（接盘池见底 / 承接枯竭）
  → 退潮
```
**两点纪律**：
- **出货 ≠ 见顶**：牛市中段也有高位天量（如 2015H1 连环天量连环新高）；灯的语义是"楼越盖越虚"，是风险计量，不是日期。
- 顶的**完成**是序列信号：**天量之后的高位量缩**=承接枯竭。这才是"缩量"唯一正确的用法——它的信息量取决于出现在天量**之后**，不是孤立快照。

### 底链（只做状态描述 → 不预测）
```
低位 × 地量
  → 【止跌观察】仅证明卖方枯竭，不预示上涨
  → 需场外【催化】（产业逻辑 / 消息）引入新钱
  → 低位放量 = 【测试】（试空方压力，自身筹码不足）→ 不追
  → 默认【震荡洗盘】（洗出浮筹、集中筹码）
  → 回踩不破 + 再放量 = 【确认】
```

## 6. 输出形态

每日产出三样，全部相对滚动基线：

1. **过热度 0–100**：五力分位的加权合成（权重少、固定、可解释）。
2. **三盏灯**：极端（资金量/价格位置分位见顶）· 背离（价新高但广度不确认）· 衰竭（下行动力抬头 / 天量后量缩）。
3. **状态标**：顶链 / 底链当前所处节点（出货灯亮 / 承接枯竭 / 止跌观察 / 测试 / 洗盘 / 确认）。

**动作语义**：三盏灯齐亮 → "退潮风险高，降仓 / 收紧止盈 / 不追高"，**不是**"明天见顶"。

---

## 7. 验证方法（别用胜率）

- **指标不是 accuracy，是条件分布右移**：灯亮之后，前向 5–20 日的**最大回撤分布**相比无条件基准是否显著右移（过热端）。
- **严格滚动样本外（walk-forward）**：用 t 之前的数据定基线与规则，验 t 之后，杜绝前视/样本内回填（cycle_quant 旧准确率正是栽在 in-sample backfill）。
- **基准要对**：方向类永远和"市场上涨基准率（实测≈55%）"比，不和 50% 比。
- **参数越少越好**：优先用分位排名而非拟合系数；过热度合成权重手定、可解释。

---

## 8. 落地可行性 & 待定

**现在就能算**（`market_data.db`）：成交额/换手、涨跌停、涨跌家数、新高新低、连板/炸板（`tushare_limit`）、高低切（`sector_daily`）、北向（`north_flow`）。顶链所需变量基本齐备。

**待定/局限**：
- "高位"的操作化（距筹码密集区/近期低点的涨幅分位 + 价格区间分位近似）需定义与调参。
- 两融余额、龙虎榜席位等"强弱手"直接证据 `market_data.db` 暂无，可后补。
- 北向实时口径自 2024 后弱化，权重宜低。
- v1 是**指数级**过热度；后续可下沉到板块级、个股级（接渊图"公司×逻辑"映射）。

---

## 9. 一句话总纲

**过热端用一个条件（高位×天量）做风险推断，冰点端用一串状态只做观察；一切相对滚动基线，复杂在特征、简单在规则；输出风险灯不输出预测。** 顶尖底圆，反者道之动——但只在能看清的那一端动作。
