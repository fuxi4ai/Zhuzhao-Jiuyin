## 一、第一性定义（Doctor 2026-06-10，照录）

情绪周期判断的是**风险偏好的周期**。把市场"风险偏好上行期"与"下行期"标注出来：

| 期 | 季节 | 定义 |
|---|---|---|
| 上行 | **春**（第一周期） | 复苏期，资金"蠢蠢欲动" |
| 上行 | **夏**（第二周期） | 火热期，资金兴致勃勃，**风险逐渐积累** |
| 下行 | **秋**（第三周期） | 衰退期，从过热开始冷却 |
| 下行 | **冬**（第四周期） | 冰点期，可能有 1~数次"**极寒**"，但也孕育新机会 |

原则：**定性难定量，但需以定量约束定性**（人为分界，机器给约束）。

**因果机制（Doctor 2026-06-10 补充）：风险偏好的周期性源于"赚钱效应"带来的 FOMO 与 fear 心理。**

```
赚钱效应出现 → FOMO（追入）→ 自我强化 → 透支（风险积累）
→ 亏钱效应 → fear（回避）→ 自我强化 → 冰点（极寒=fear 高潮）
→ 赚钱效应重新萌芽 → …（反身性循环，~2 个月一轮）
```

四季的机制学含义：春=赚钱效应萌芽（先觉者入场）｜夏=FOMO 自我强化（追入者抬轿，风险积累）｜秋=亏钱效应显现（追入者被套，fear 滋生）｜冬=fear 自我强化至冰点（极寒=恐慌高潮，亦即卖压出清=新机会孕育）。
与过热度模型自洽：火热点贴近高点，正因 FOMO 高潮=弱手接力枯竭=强手高位交货完成。

**单向性铁律（Doctor 2026-06-10 四次对齐）**：公众风险偏好的 FOMO/fear 是循环的（CO 利用公众的风险偏好）——**春夏秋冬单向前进，不能反复**。春→夏→秋→冬→春（新周期 N+1）。
**报错语义**：若已报"二转三"（夏→秋）而市场又火热，**那是秋报错了**——撤回该次转换、修正历史标注（标 corrected 留痕），而不是"市场退回第二周期"。引擎实现=单向状态机 + 修正触发器：
- 秋/冬期间评分突破本周期夏段峰值 → 撤回秋/冬宣告，本段回标夏（FOMO 未死）；
- 春/夏（新周期）期间评分跌破上周期冬段谷值 → 撤回春宣告，回标冬（fear 未尽，周期号回退）；
- 未触发修正的反向波动 = 段内反弹/回踩，只注记不改季。
副产品：单向性使**周期编号**成为可能（冬→春即 N+1），日报"第几周期"字段就此有了体系内正源。

|---|---|---|---|

新增成分仅一项：**晋级率**（昨日涨停股今日连板表现）——小鲍现行综合法的核心项，cycle_quant 缺失，需补拉 `limit_list_d`（#4 已批）。连板高度保留但降权（24-09 失灵教训）。

**成分的因果分层**（由 FOMO/fear 机制推出，影响先行性预期与调参方向）：

| 层 | 成分 | 机制角色 |
|---|---|---|
| **因·赚钱效应直测** | 晋级率、（可加）涨停股次日溢价 | 昨天追的人今天赚没赚——FOMO/fear 的直接燃料，**预期最先行** |
| 果·行为症状 | 涨停数、跌停数、涨跌比、主线宽度 K_day | 心理已转化为行动后的足迹 |
| 果·滞后症状 | 成交额、连板高度 | 群体行为的尾部计量，确认性强、先行性弱 |

调参含义：§3.4 校准时若"因"层权重自然胜出，机制得到数据支持；若"果"层胜出，提示机制假设需复检——权重结果本身就是机制的检验。

### 3.2 四季判定（在 emotion_cycle 既有字段上落 Doctor 第一性定义）

```
total_score（滚动分位化后）的趋势项定上/下行，水平项定季节：
上行 × 低位→中位 = 春 ｜ 上行 × 高位 = 夏
下行 × 高位→中位 = 秋 ｜ 下行 × 低位 = 冬
极寒 = 冬内单日极端事件标记（口径待问题 #2）
```

**定位铁律（Doctor 2026-06-10 三次对齐）：情绪周期是参考指标，不是交易策略——必须"先行"提示风险/机会，不要滞后确认。** 确认期是交易指标防假信号用的；定性预测框架判错了改标注即可，先行价值 > 准确率。

落地形式：每日输出 = **当前季节 + 前瞻倾向**，两个都是先行口径：
- 夏·高位 → 提前点"秋风险积累中"（不等顶部确认）；
- 冬·极寒后 → 提前点"春机会孕育中"（不等右侧放量）——对应第一性定义"冬也孕育着新的机会"。

**顶底不对称的正确用法**（修正：作用于**置信度**而非时滞）：顶侧先行提示置信度高（单条件推断），底侧先行提示置信度低（多条件过程、证据天然不足）——两侧都先行说出来，只是标注的把握度不同。判定全侧当日即出，事后翻改留痕、不设确认门。

机器输出季节+前瞻倾向+置信+依据成分 → 人（Doctor/九儿）终裁，override 留痕。

### 3.3 周期时长先验（Doctor 2026-06-10 经验值，"不一定准确"——经验非规律）

**全周期（春→冬）约 2 个月循环一次，区间 1~3 个月。** 由此锁定全部窗口参数：

| 参数 | 取值 | 推导 |
|---|---|---|
| 全周期典型长度 | ≈40 交易日（区间 20~65） | 2 个月（1~3 月） |
| 单季典型长度 | ≈10 交易日（5~15） | 全周期 ÷ 4 |
| 滚动分位基线窗 | 40~60 交易日（≈1 个周期） | 太长跨周期摊平、太短被季内噪音绑架 |
| 趋势均线 | 短 3~5 日 / 长 10~15 日 | 需分辨 ~10 日的季相 |
| 异常哨兵 | 单季 >30 交易日 或 全周期 >65 交易日 → 提示"周期拉长/范式变化?" | 先行提示，不强制换季 |

回检：recap_daily.cycle_number 库内仅 1 条（小鲍周期编号未成体系落库），先验暂无法用历史标签验证——重算跑出 2025-10 以来的季节序列后，用实际周期长度分布回检此先验，偏离过大则参数重调。

### 3.4 校准目标函数（Doctor 2026-06-10 裁定，回答原问题 #5）

**微调目标：让"火热点"（夏→秋交界）尽量贴近市场短期最高点，"极寒点"尽量贴近市场短期最低点。** 市场在适应和变化，尽力就好——不追求完美拟合，防过拟合优先。

落地口径：

| 要素 | 口径 |
|---|---|
| 市场短期极值 | 上证（或主线加权）±10 交易日分形高/低点（窗口与 ~40 日周期先验匹配） |
| 距离度量 | 夏秋交界日 ↔ 最近局部高点、极寒日 ↔ 最近局部低点 的交易日距离 |
| 评分 | **非对称**：先行 0~5 日最优（参考指标的先行使命），滞后罚重于先行；命中率（±3 日内）+ 中位距离双指标 |
| 调参对象 | 成分权重（等权起步）+ 分位窗/均线参数（§3.3 框定范围内微调） |
| 防过拟合 | 样本期仅 2025-10 起、一个弱市区间——参数只做粗调（如权重 0.5 步长），不做精细优化；**季度滚动复核**，接受市场适应导致的缓慢失效（过热度模型宪法前提 2） |

至此原五问全部闭环：#1 时效=全侧先行 ✅ #2 极寒=跌停>涨停且>10家+分位定强度 ✅ #3 过渡态=前瞻倾向字段 ✅ #5 权重=本目标函数调 ✅ #4 limit_list_d 已回填（345 日全覆盖，顺手修复涨停口径断裂 GOTCHA）✅。
