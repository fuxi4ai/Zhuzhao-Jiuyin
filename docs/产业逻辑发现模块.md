## 🎯 信息差评分 (info_gap_level)

> **核心认知（2026-06-10 校准修订）：** 越明确 + 越直接 + 传播越广 → 信息差越小 → **超额收益幅度越小**。
> ~~信息差越大→窗口期越长~~ **已被 ETF 级回测证伪**：板块启动节奏与 gap 无关（中位 lag 平在 ~10 个交易日），gap 决定的是涨幅天花板，不是潜伏时长。**gap 大≠可以慢慢建仓。**

| 等级 | 含义 | 特征 | 信号数 | 典型案例 |
|------|------|------|--------|---------|
| **1** | 最小信息差 | 明确 + 直接 + 广传播 | 156条 (17.2%) | 碳酸锂20万（具体数字+直接涨价+全市场热议） |
| **2** | 较小信息差 | 较明确 + 较直接 + 较广传播 | 614条 (67.8%) | 英伟达GTC大会（有数字+直接+行业关注） |
| **3** | 中等信息差 | 中等模糊 + 中等传导 + 中等传播 | 117条 (12.9%) | 海外数据中心电力需求（逻辑链2层+中等传播） |
| **4** | 较大信息差 | 模糊 + 间接 + 窄传播 | 18条 (2.0%) | COWOP封装演进（逻辑链3层+小众环节） |
| **5** | 最大信息差 | 极模糊 + 多层推理 + 极窄传播 | 1条 (0.1%) | 联讯仪器光检测（逻辑链3层+"全球唯二"+极窄环节） |

### 评分维度

| 维度 | 权重 | 1分 | 5分 |
|------|------|-----|-----|
| 明确度 | 15% | 有具体价格数字 | 无数字+模糊词 |
| 直接性 | 60% | 直接受益 | 多层传导链+小众逻辑 |
| 传播广度 | 25% | 全市场热议 | 极窄环节(检测/代工/中间体等) |

### 联讯仪器案例（info_gap_level = 5）

**信号内容：**
> 光模块速率从400G向1.6T演进，检测需求成为卡点；联讯仪器为全球唯二攻克1.6T测试方案的厂商，无论CPO还是NPO路线检测需求均确定增长

**评分分解：**
| 维度 | 得分 | 说明 |
|------|------|------|
| 明确度 | 3 | 无具体数字，有"无论...均"模糊描述 |
| 直接性 | 5 | 3层传导链(光模块演进→检测卡点→联讯受益)+"无论...均"+"卡点"小众逻辑 |
| 传播广度 | 5 | "检测/代工/测试"3个冷门关键词，极窄传播 |
| **综合** | **5** | **信息差最大，超额幅度天花板最高**（实测峰值超额 25.6%、lag 7 天=全表最快；"潜伏窗口最长"原假设已证伪） |

**对比：碳酸锂20万（info_gap_level = 1）**
| 维度 | 联讯仪器 (5) | 碳酸锂20万 (1) |
|------|-------------|--------------|
| 明确度 | 无具体数字 | 具体价格"20万/吨" |
| 直接性 | 3层传导+"无论...均" | 直接涨价 |
| 传播广度 | 检测环节极窄 | 全市场热议 |
| 价格反应 | 板块启动同样快（lag 7 天），但超额幅度高且持续 | 当日涨停，次日分化 |
| 超额收益 | 幅度天花板高（峰值超额 25.6%） | 幅度有限（中位峰值超额 10.9%），追高风险大 |

### 📐 数据校准（2026-06-10，ETF 级，426 组去重样本 + yuantu 20 条）

| gap_level | n | 兑现率 | 中位 lag | 中位峰值超额 |
|---|---|---|---|---|
| 1 | 53 | 98% | 10 天 | 10.9% |
| 2 | 277 | 97% | 10 天 | 13.0% |
| 3 | 83 | 99% | 11 天 | 14.2% |
| 4 | 12 | 100% | 8.5 天 | 17.9% |
| 5 | 1 | 100% | 7 天 | 25.6% |

**三结论：** ① 峰值超额随 gap_level 严格单调递增 → info_gap_level 可作期望收益/仓位权重先验（gap≥3 显著更高）；② lag 与 gap 无关 → 窗口假设证伪；③ 120 日兑现率 ~98% 无区分度 → "是否兑现"不是触发器，date_realized（何时启动）+ excess_peak（涨多少）才是。
**局限：** level 4/5 仅 13 条样本；同主线信号共享 ETF 超额曲线，lag 测的是板块启动而非信号级认知——标的级窗口期待 ticker_resolver 补全后复测。
**口径出处：** `closure_engine.py`（Y=3/X=5%/绝对回撤 5pp）· `兑现检测_审核表_20260610.tsv`

## 🔎 使用方法

### CLI 工具
```bash
cd projects/烛照九阴/tools/
python3 logic_discovery.py <command>
```

| 命令 | 功能 | 示例 |
|------|------|------|
| `types` | 列出所有逻辑类型 | `python3 logic_discovery.py types` |
| `by-type TYPE` | 按类型查询 | `python3 logic_discovery.py by-type supply_shock` |
| `by-sector SECTOR` | 按行业查询 | `python3 logic_discovery.py by-sector 碳酸锂` |
| `hot` | 高价值信号 | `python3 logic_discovery.py hot` |
| `trend` | 月度趋势 | `python3 logic_discovery.py trend` |
| `timeline TYPE` | 时间线 | `python3 logic_discovery.py timeline demand_surge` |
| `stats` | 统计摘要 | `python3 logic_discovery.py stats` |
| `search KEYWORD` | 关键词搜索 | `python3 logic_discovery.py search 霍尔木兹` |

### SQL 查询
```sql
-- 查询所有供给冲击信号
SELECT date, category, keyword, signal_content, info_gap_level
FROM industry_signals 
WHERE logic_type = 'supply_shock'
ORDER BY info_gap_level DESC;

-- 最大信息差信号（潜伏价值高）
SELECT date, category, keyword, signal_content, info_gap_level
FROM industry_signals
WHERE info_gap_level IN (4, 5)
ORDER BY date DESC;

-- 最强信号：供给冲击 + 涨价 + 高信息差
SELECT date, category, keyword, signal_content, target, info_gap_level
FROM industry_signals
WHERE logic_type IN ('supply_shock', 'price_driven')
  AND info_gap_level >= 3
  AND (signal_content LIKE '%涨价%' OR signal_content LIKE '%缺口%')
ORDER BY date DESC;

-- 行业 × 逻辑类型交叉统计
SELECT category, logic_type, count(*) as cnt
FROM industry_signals
GROUP BY category, logic_type
ORDER BY cnt DESC;
```

## 数据库结构

`industry_signals` 表新增字段：
- `logic_type` TEXT — 产业逻辑类型（tech_innovation/supply_shock/demand_surge/price_driven/event_driven/emotion_cycle/other）
- `info_gap_level` INTEGER — 信息差评分（1-5，1=最小，5=最大）

参考表 `logic_type_mapping` 存储中英对照和描述。
