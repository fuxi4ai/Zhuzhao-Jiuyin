## 1 · 项目概述

**定位**: 个人复盘与市场分析系统，整合小鲍四维度训练营课件 + 老版实操表 + 外部数据源

**核心目标**:
- 🎯 **丰满数据**：不断扩充数据库记录数和字段填充率
- 🎯 **回测驱动**：当库中历史数据能指导下一天板块/大盘走势时即为成功
- ⚠️ **回测约束**：严格避免未来函数（look-ahead bias）

**核心能力**:
- 📊 四维度框架：外围定价 → 行业主线 → 情绪技术 → 交易策略
- 📈 量化情绪周期：6指标综合评分 + 天哥直觉双轨对比（一致率 93.2%）
- 🔍 信号回测：历史信号验证（121信号，胜率 54.5%）
- 🎯 错题本：交易错误追踪与复盘

**关键指标**:
- 复盘记录：351条 → 交集范围128条
- 量化情绪周期：533条 → 交集103条
- 重点标的：219条（30只个股）
- 信号回测：121条
- 错题本：10条（9已改正，1处理中：G007）

- 日期格式: `YYYY-MM-DD`，必须在合理范围内

## 11 · 顶层设计迭代（2026-05-11）

### 11.1 烛照九阴的本质

**不是做什么：**
- ❌ 发现好用的技术图形
- ❌ 跟随外围市场的阶段性投机

**而是做什么：**
> **秉真知之烛，照九阴之地**
> 
> 发现尚未被市场充分定价的信息差，在产业逻辑与市场认知的交汇处寻找超额收益。

### 11.2 两条主线

#### 主线1：市场热度与情绪周期

| 层级 | 内容 | 数据 |
|------|------|------|
| **数据层** | 统计指标 | 涨价家数比 / 涨停数 / 跌停数 / 封板率 / 沪深总成交额 |
| **分析层** | 优化版情绪周期 | 当前处于哪个阶段（冰点→修复→主升→高潮→退潮） |
| **产出** | 情绪周期定位 | 指导仓位管理和买卖时机 |

#### 主线2：底层产业逻辑 vs 市场认知差

| 层级 | 内容 | 举例 |
|------|------|------|
| **发现层** | 技术迭代 → 供需变化 → 利润预期 | CoWoS 产能 97% → 封装瓶颈 → 哪些企业受益 |
| **追踪层** | 市场热点板块涨了多少天 | 新闻是否跟上了产业逻辑？市场认识到哪一步了？ |
| **核心** | 信息差 = 产业逻辑已知但市场尚未认知的窗口期 | 200G 硅光良率爬坡中，市场还在炒 800G |

**底层逻辑链条：**
```
产业逻辑（技术迭代/供需变化）
    ↓ 影响
企业利润变化
    ↓ 改善
投资者预期
    ↓ 形成
信息差 → 超额收益（具备确定性底层逻辑）
    ↓ 共振条件
市场热点认知 × 地缘政治 × 大宗商品 × 资金动向
```

### 11.3 行为模式识别（辅助主线）

复盘帮助识别机构和游资的行为规律：
- 大盘连涨 N 天后可能震荡回调
- 机构对特定类型地缘政治风险的反应模式
- 游资对新闻炒作的节奏规律

### 11.4 新闻模块重新定义

**不应该做的事：**
| 旧做法 | 问题 |
|--------|------|
| 研报标题堆砌 | 已被市场定价的噪音 |
| 行业周报入库 | 泛泛而谈，无信息差 |
| 大模型 KB 摘要 | 缺乏提炼，无法指导决策 |
| 什么新闻都抓 | 噪音淹没信号 |

**应该做的事：**
| 新方向 | 说明 |
|--------|------|
| 捕捉信息差 | 专家纪要里的产能/良率/扩产具体数据点 |
| 追踪产业逻辑 | 供需缺口变化、技术路线切换的时间线 |
| 记录共振信号 | 地缘事件 + 大宗商品 + 资金流向的交叉点 |
| 沉淀行为模式 | 机构/游资对特定事件的反应规律 |

**新闻模块的核心价值：不是"数据抓取"，而是"温差测量"——产业逻辑（九儿发现的）和市场热点（新闻炒的）之间的时间差就是超额收益的来源。**

### 11.5 数据入库新标准

| 标准 | 说明 |
|------|------|
| ✅ 有具体数字 | 产能 X 万片、良率 Y%、价格 Z 元 |
| ✅ 能改变供需格局 | 技术突破/产能瓶颈/需求爆发 |
| ✅ 市场尚未充分认知 | 信息差存在 |
| ❌ 泛泛而谈 | "看好 AI 产业链"、"建议关注" |
| ❌ 已被定价 | 研报推荐、新闻头条 |

## 12 · 复盘框架重构（2026-05-12）

### 12.1 数据库重构

**旧 schema 问题：**
- 4 个维度表 + recap_daily，字段过多
- Dim2/Dim3 堆积了大量低质量研报标题和泛泛周报
- 95% 是噪音，5% 是有价值的信息差

**新 schema（极简）：**

| 表 | 主线 | 用途 |
|---|------|------|
| `emotion_cycle` | 主线1 | 情绪周期：涨跌停/封板率/成交额 → 季节 → 仓位建议 |
| `industry_signals` | 主线2 | 产业信号：哥哥的词头（技术升级/供给紧缺/需求爆发） |
| `hot_sectors` | 辅助 | 热点板块：ths_daily Top 5 |
| `information_gap` | 主线2 | 信息差追踪：发现→市场认知→兑现 → 买卖信号 |
| `daily_summary` | 综合 | 每日复盘摘要（极简版） |

**旧表标记废弃但保留兼容：**
- dim1_external_pricing → 废弃
- dim2_sector_themes → 废弃（数据已清理，只留有具体数字的条目）
- dim3_sentiment_tech → 废弃（同上）
- dim4_trading_strategy → 废弃
- recap_daily → 废弃

### 12.2 复盘流程

