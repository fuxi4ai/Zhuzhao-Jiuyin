# 新闻信源方案（收窄 · 2026-06-03 定）

> 财新已娱乐化、价值偏低 → **搁置备查**。当前信源**收窄为单源**：**小鲍老师每日复盘课件**里整理的新闻 / 产业逻辑语料，**置信度统一 P2**（复盘老师解读，非一手源；沿用渊图 P 分级）。

## 信源与流水
```
小鲍每日复盘课件（Raw-Recap/ + 每日新课件）
   → xiaobao_extractor:
       · 市场数据 → 正则抽（指数/成交额/涨跌…）
       · 新闻/产业逻辑 → 九儿(Claude)自己抽（不调百炼）
   → 落**复盘库 `Database/烛照九阴/recap.db`**（市场数据→dim1_external_pricing/recap_daily；行业/产业逻辑→dim2_sector_themes/industry_signals），带 source="小鲍复盘课件" · confidence="P2"
   → 复盘四维度（尤其 Dim2 行业主线）引用
```
> **目标库钉死**：复盘库 `Database/烛照九阴/recap.db`。**不是**渊图 `Database/行业研究/…knowledge_graph.db`，**也不写** `news.db`。

## 句芒要做的（薄改造）
1. `xiaobao_extractor` 的"产业逻辑 LLM 抽取"步：去百炼，改由**九儿现场做**（脚本只留正则抽市场数据 + 落库接口）。
2. 入库记录加 `source` / `confidence(P2)` 字段（news_events 或 recap 对应表），便于复盘时按置信度加权。
3. 旧桥 `caixin_bridge.py` / `gangtise_bridge.py` 标 deprecated（不再取数）。

## 每日入库（已定 · 2026-06-03）
你把当天小鲍课件丢进 `Database/烛照九阴/Raw-Recap/`，两道定时工序自动处理（都在静默时段、不打扰、留日志）：
- **23:00 · 九儿入库**（`recap-kejian-daily-ingest`）：扫 Raw-Recap 新课件 → 市场数据正则抽 + 产业逻辑九儿自抽（不调百炼）→ **严格去重**（`processed_kejian` 表，filename+md5 双判）→ 落 recap.db/news_events，带 `source=小鲍复盘课件 / confidence=P2`。
- **23:30 · 句芒审核**（`recap-kejian-review`）：复核去重完整性、数据合理性、P2 标签、归位合规 → 出审核日志，只报不改。
- 30 分钟间隔为缓冲；有问题次日早上看日志（`brain/agents/烛阴/logs/` 入库、`brain/agents/句芒/logs/` 审核）。

## 财新方案（搁置 · 备查）
原拟用 Claude in Chrome 抓「财新数据通」VIP（见本目录早期方案）。因娱乐化暂不启用；若日后要补一手新闻源，可重启该路（浏览器自动化 + 九儿清洗），届时财新标 P1-C/P2。

## P 分级备忘
P1-A 一手源（年报/公告/官方）… P2 二手/解读/自媒体。小鲍复盘 = **P2**：可用于情绪/产业线索，但结论须经更高阶源或数据交叉，不作硬事实。
