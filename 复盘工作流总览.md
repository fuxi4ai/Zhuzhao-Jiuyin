## 晨间定时链（顺序即依赖）

```
06:00  课件入库      recap-kejian-daily-ingest   九儿  → recap.db（管线③）
06:15  公共行情更新  market-data-daily-update    句芒  → market_data.db 公共表（管线①）★必须早于07:00
06:30  课件审核      recap-kejian-review         句芒  → 审核日志（只读核管线③）
07:00  四表+日报+Artifact  zhuzhao-market-fetch-daily-report  烛阴 → 管线②+情绪引擎④+出日报+同步侧栏
```

**关键顺序铁律**：日报（07:00）依赖①公共行情的市场宽度（stock_daily 涨跌家数）。故①必须排在07:00之前（2026-06-14 由08:00提前到06:15，修「日报读到滞后宽度」根因）。

## 各线 vintage 怎么读

- 报表里每个区块带 vintage 角标，四条线 vintage 可以不同、按各自最新如实显示，缺数诚实标注、绝不编。
- 小鲍 band（总仓位）取 `dim4_trade_plan` 最后一个非空 `position_band`。**dim3 情绪 + dim4 结构化 band 是 2026-06-10 才加的列**——此前入库的课件无这些列，会造成 band/dim3 历史断档；要连续需强制重处理补列（见下「补缺范式」）。

## 补缺范式（历史课件回填结构化列）

课件去重按 filename+md5；哈希不变不会自动重跑。要给旧课件补 dim3/dim4 结构化列，需**强制重处理**指定日期区间，只补空列、不重插 dim1/dim2。范式：备份 recap.db → 子代理逐篇读 PDF 提炼 dim3/dim4 → 句芒口径自检（归一/词表/只增不减）→ 重生成日报。
