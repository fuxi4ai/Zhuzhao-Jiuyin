## 晨间定时链（顺序即依赖）

```
06:00  课件入库      recap-kejian-daily-ingest   九儿  → recap.db（管线③）
06:15  公共行情更新  market-data-daily-update    句芒  → market_data.db 公共表（管线①）★必须早于07:00
06:30  课件审核      recap-kejian-review         句芒  → 审核日志（只读核管线③）
07:00  四表+外盘+派生回填+日报+Artifact  zhuzhao-market-fetch-daily-report  烛阴
        → 管线②四表 + ②b外盘指数(fetch_intl_index) → 5.x 派生回填(④情绪 + ⑤渊图入库/gap/closure auto-apply/标的池, recap.db 同走 /tmp)
        → 出日报 + 同步侧栏
```

**关键顺序铁律**：
- 日报（07:00）依赖①公共行情的市场宽度（stock_daily 涨跌家数）。故①必须排在07:00之前（2026-06-14 由08:00提前到06:15，修「日报读到滞后宽度」根因）。
- 07:00 任务内部顺序（2026-06-23 三线核实后定）：②四表抓取 → ②b外盘指数(fetch_intl_index) → 守门验 stock_daily 就绪 → recap.db 留 predaily 锚 → emotion_engine_v2 --apply → sync_buy_signals → gap_rater → closure_engine **--apply** → populate_signal_targets → gen_daily_report。情绪/渊图/兑现都必须在出报前回填，否则日报读到旧状态。
- **closure 自动 apply（2026-06-23 Doctor 裁定，改自原 dry-run 门控）**：gap_status 转移是 Doctor 已拍板的确定性公式、非 CC 判断，故自动 apply 不违「CC 不自动打✓」铁律。安全垫：每日先留 `recap.db.bak_{日}_predaily` 锚 + 变更 diff 落 `docs/兑现变更_{日}.md` + 通知，Doctor 事后抽查，错了一条命令回滚。

- 报表里每个区块带 vintage 角标，四条线 vintage 可以不同、按各自最新如实显示，缺数诚实标注、绝不编。
